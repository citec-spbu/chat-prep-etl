# Инструмент подготовки данных «Очиститель мессенджеров» и исследование его влияния на эффективность AI-ассистента

## Описание проекта

Chat Preparation ETL — это инструмент для обработки, очистки и обезличивания историй переписок из мессенджеров, а также система тестирования влияния качества данных и prompt engineering на эффективность AI-ассистента.

Проект предназначен для:
- обработки Telegram-чатов и экспортов переписок;
- очистки и anonymization текстовых данных;
- подготовки данных для LLM/RAG-систем;
- тестирования AI-ассистентов на различных типах данных и prompt-стратегиях;
- анализа влияния cleaning pipeline на качество ответов языковых моделей.

---

# Цели проекта

- Разработать ETL-систему для обработки Telegram-чатов.
- Реализовать anonymization персональных данных.
- Подготовить данные для подачи в языковую модель.
- Создать систему тестирования AI-ассистентов.
- Реализовать сравнение raw и cleaned data.
- Измерить влияние cleaning pipeline на качество ответов модели.

Проект построен по принципам Clean Architecture и разделен на два независимых сервиса.

## 1. Сервис очистки данных (Cleaner Service)

Cleaner Service предназначен для обработки данных мессенджеров. Сервис включает:
- загрузку и парсинг экспортов Telegram-чатов;
- очистку текстовых данных;
- anonymization персональной информации;
- подготовку сообщений для дальнейшего использования в LLM/RAG-системах и векторных базах данных.

Основной доменной сущностью сервиса является `MessageMetadata`.

```python
MessageMetadata:
- chat_id: int
- sender_id: int
- text: str
- attached_files: str
```

Именно в этом формате сообщения проходят через все этапы ETL pipeline.
После обработки данные преобразуются в embeddings и сохраняются в vector database для semantic search.

---

## 2. Сервис тестирования AI-систем (Testing System)

Testing System предназначена для тестирования различных сценариев работы модели, сравнения качества ответов на raw и cleaned data, а также расчета и анализа метрик качества.

Основной сущностью системы является `Experiment`.

```python
Experiment:
- id: str
- name: str
- config: str
- questions: List[Question]
- answers: List[Answer]
- metrics: List[MetricValue]
- started_at: datetime
- finished_at: datetime
```

Сервис позволяет сравнивать работу моделей в четырех сценариях:

1. Dirty data + basic prompt  
2. Clean data + basic prompt  
3. Dirty data + improved prompt  
4. Clean data + improved prompt  

Для удобства использования реализован frontend на Streamlit, позволяющий:
- запускать ETL pipeline;
- загружать чаты;
- выполнять semantic search;
- запускать AI-эксперименты;
- просматривать результаты тестирования через графический интерфейс.

# Метрики системы

Для оценки качества AI-ассистента используются следующие метрики:

1. **Latency** — метрика времени, фиксирующая скорость ответа модели на пользовательский запрос.

2. **Token Count** — метрика количества токенов, использованных моделью при генерации ответа.

3. **BERTScore** — метрика семантического сходства между ответом модели и эталонным ответом на основе BERT-эмбеддингов.

4. **Exact Match** — метрика точного совпадения ответа модели с эталонным ответом:
- `1` — ответы совпадают;
- `0` — ответы различаются.

5. **Jaccard Distance** — метрика различия между текстами на основе пересечения уникальных слов.

6. **ROUGE-L** — метрика сходства между ответом модели и эталонным ответом на основе совпадения последовательностей слов.

7. **Hallucination Score** — метрика, оценивающая наличие в ответе модели информации, отсутствующей в исходном контексте.

8. **Numeric Accuracy** — метрика точности числовой информации в ответе модели по сравнению с эталонным ответом.

9. **Overlap Metrics** — группа метрик оценки пересечения слов между ответом модели и эталонным ответом:
- Precision — точность совпадения;
- Recall — полнота совпадения;
- F1-score — общий показатель качества совпадения.
